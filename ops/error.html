ops.error — Contains exception classes for operation errors¶

Contains a number of exception classes for errors that can occur in the ops package.

All exception classes in this module extend util.ApplicationError.

Value-related Errors¶

The exception classes in this section also extend ValueError.

class ops.error.InvalidHeatingCurrentError¶
Raised when one of the various startHeatingWithCurrent() methods is called with an invalid current.
class ops.error.InvalidTargetTemperatureError¶
Raised when ProductionSystem‘s startHeatingToTemperature() method is called with an invalid heating temperature.
class ops.error.InvalidHeaterPositionError¶
Raised when one of the various startHeaterMovement() methods is called with an invalid target position.

State-related Errors¶

class ops.error.NotCalibratedError¶
Raised by various methods of ProductionSystem and CalibrationData if they are called while the system has not yet been successfully calibrated.
class ops.error.RequiresSimulationError¶
Raised by ProductionSystem when an operation that requires a SimulatedDeviceInterface is performed on a system that wraps a real DeviceInterface.

Locking-related Errors¶

class ops.error.LockingError¶
The base class for SystemLockedError, SystemNotLockedError, and WrongKeyError.
class ops.error.SystemLockedError¶
Raised when a client tries to start an operation on a locked ProductionSystem without providing a key, or tries to lock a ProductionSystem that is already locked.
class ops.error.SystemNotLockedError¶
Raised when a client provides a key when starting an operation on a ProductionSystem that is not locked, or tries to unlock a ProductionSystem that is not locked.
class ops.error.WrongKeyError¶
Raised when a client provides the wrong key when starting an operation on or unlocking a locked ProductionSystem.
